Alua Kernel

A Pure, Independent OS Kernel Environment.
核心定義：
Alua Kernel 是完全脫離 EFI 依賴（除引導程序 Bootloader 之外）的獨立內核。它旨在提供一個更純粹的執行環境，不直接調用韌體運行時服務。

目前開發進度

核心狀態：引導程序已成形 (Bootloader Ready)

整體開發進度：15%

設計目標

  • 硬體抽象化：不再依賴 UEFI Runtime Services，改為直接控制硬體中斷與內存分頁。
  • 高性能調度：針對 Alua 腳本引擎優化的內核級任務調度。
  • 模組化驅動：支持動態加載內核模組 (KM)。

技術說明

與 EFI 版本不同，Alua Kernel 在接管 CPU 控制權後會切換至保護模式或長模式，並重新構建中斷描述表 (IDT) 與全局描述表 (GDT)。

與 EFI 版的差異？

EFI 版是運行在韌體環境下的 App，而 Kernel 版則是試圖成為「環境本身」。如果您需要開發底層驅動或作業系統組件，Kernel 版將是未來的首選。
