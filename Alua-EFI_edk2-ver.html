Alua (Alua Engine)

目前版本：0.4.5
A High-Performance, Register-based Scripting Engine for UEFI Environments.

Alua 是一個專為 AOS (Alua Operating System) 啟動環境開發的腳本引擎。它保留了 Lua 簡潔的語法特性，但針對底層韌體環境進行了深度重構與優化。

核心技術

  • Firmware-Native ABI: 核心完全基於 UEFI MS ABI 構建，與 edk2 深度整合，已補足標準 C 庫 (LibC) 支持。

註：開發進度極深，目前正在評估移植方案。

使用方法

建議使用 QEMU 測試，請務必等到 1.0.0 正式版 後再於實體機器進行測試。

git clone https://github.com/JW0521-X/Alua.git
cd Alua
mkdir qemu_root
cp -r ./ROOTFS/* ./qemu_root
cp EFI/BOOT/BOOTX64.EFI ./qemu_root/EFI/BOOT

qemu-system-x86_64 -machine q35 -accel tcg -cpu max -pflash /path/to/ovmf.fd -drive format=raw,file=fat:rw:qemu_root -m 256

常見問題 (FAQ)

EFI 版為什麼不會被廢除？

因為 rootfs 深度依賴 UEFI 提供的 API，大幅修改將導致系統結構崩潰。

各版本 App 互不相容嗎？

取決於開發者是否提供跨平台 API (km/so/efiP)。若僅提供特定模塊（如 powermanager），則其他環境將無法啟動。

注意：請勿嘗試自行編寫 App API，除非你完全理解其運作機制。

舊版本會被刪除嗎？

不會。所有版本均會保留供測試與研究，但 EOL 版本不保證運行穩定。
